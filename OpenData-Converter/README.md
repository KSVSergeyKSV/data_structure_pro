# OpenData Web Converter (ODWC)

Автономный веб-конвертер открытых данных для подготовки пакетов файлов для портала data.gov.ru.

## Версия
2.0 (соответствует Методическим рекомендациям Минцифры России v4.0)

## Описание

OpenData Web Converter - это локальное одностраничное веб-приложение (SPA), которое работает полностью в браузере без необходимости установки и без отправки данных на сервер.

### Ключевые особенности

- **Полная автономность**: Работает без интернета, все вычисления выполняются в браузере
- **Безопасность**: Файлы не покидают компьютер пользователя
- **Кроссплатформенность**: Работает в любом современном браузере (Chrome, Firefox, Edge, Yandex)
- **Не требует установки**: Просто откройте index.html в браузере
- **Соответствие МР 4.0**: Генерирует файлы согласно методическим рекомендациям

## Структура дистрибутива


## Быстрый старт

1. Скопируйте папку `OpenData-Converter` на локальный диск
2. Откройте файл `index.html` двойным кликом (или через браузер)
3. Создайте новый раздел (профиль пользователя)
4. Следуйте мастеру из 5 шагов

## Функциональность

### Шаг 1: Реестр наборов данных
- Загрузка внутреннего реестра (Excel/CSV)
- Интеллектуальный маппинг колонок
- Поиск и фильтрация наборов данных
- Автозаполнение паспорта данными из реестра

### Шаг 2: Загрузка файла данных
- Поддержка форматов: XLSX, XLS, CSV, TXT, JSON
- **Опциональная транслитерация заголовков по ГОСТ 7.79-2000 (система Б)**
- Предпросмотр и ручная коррекция заголовков
- Проверка на дубликаты имён полей

### Шаг 3: Валидация структуры
- Автоматическое определение типов данных (integer, float, date, string)
- Анализ заполненности полей
- Определение максимальной длины значений

### Шаг 4: Паспорт набора данных (МР 4.0)
- Автозаполнение 19 полей стандарта
- Контекстные подсказки для каждого поля
- Генерация машиночитаемого (meta.json) и человекочитаемого (meta.html) паспортов

### Шаг 5: Экспорт пакета файлов
Генерация ZIP-архива со следующими файлами:

| Файл | Формат | Кодировка | Особенности |
|------|--------|-----------|-------------|
| data.csv | CSV | UTF-8 с BOM | Разделитель `;`, экранирование по RFC 4180 |
| structure.csv | CSV | UTF-8 с BOM | Описание структуры полей |
| structure.json | JSON | UTF-8 | Машиночитаемая структура |
| meta.json | JSON | UTF-8 | Паспорт набора данных |
| meta.html | HTML | UTF-8 | Человекочитаемый паспорт |
| report.log | TXT | UTF-8 | Журнал операций |

## Управление профилями

Приложение поддерживает многопользовательский режим через localStorage:

- **Создание профиля**: ФИО, должность, контакты
- **Экспорт профиля**: Сохранение в JSON-файл для переноса между ПК
- **Импорт профиля**: Загрузка ранее сохранённого профиля
- **Изоляция данных**: Каждый пользователь работает со своим разделом

## Транслитерация заголовков

Приложение поддерживает опциональную транслитерацию заголовков столбцов:

- **Режим по умолчанию**: Без транслитерации (кириллические заголовки сохраняются)
- **Опция ГОСТ 7.79-2000 (система Б)**: а→a, ж→zh, щ→sch и т.д.
- **Предпросмотр**: Таблица соответствия исходных и транслитерированных заголовков
- **Ручная коррекция**: Возможность отредактировать любое имя перед конвертацией
- **Валидация**: Проверка на уникальность имён после преобразования

> ⚠️ **Важно**: Содержимое ячеек данных НИКОГДА не транслитерируется.

## Требования к среде

- Любой современный браузер с поддержкой ES6+
- Разрешение экрана от 1280px
- Включённый JavaScript
- Поддержка localStorage

### Поддерживаемые ОС

- Windows (7 и выше)
- Linux (Astra Linux, РЕД ОС, Ubuntu, Debian)
- macOS

## Безопасность

- **Отсутствие сетевого кода**: Приложение не выполняет внешних запросов
- **Обработка в памяти**: Файлы обрабатываются в sandbox браузера
- **Санитизация данных**: Защита от XSS при отображении
- **Логирование**: Все действия фиксируются в журнале профиля

## Логирование и аудит

Приложение фиксирует следующие события:

- `APP_STARTED` - Запуск приложения
- `PROFILE_CREATED` - Создание профиля
- `PROFILE_SELECTED` - Выбор профиля
- `REGISTRY_LOADED` - Загрузка реестра
- `DATASET_SELECTED` - Выбор набора данных
- `SOURCE_DATA_LOADED` - Загрузка файла данных
- `TRANSLITERATION_USED` - Использование транслитерации
- `CONVERSION_STARTED` - Начало конвертации
- `PASSPORT_GENERATED` - Генерация паспорта
- `ARCHIVE_GENERATED` - Создание ZIP-архива

## Обновление

Для обновления приложения:

1. Скачайте новую версию дистрибутива
2. Замените файлы в папке `OpenData-Converter` (кроме данных пользователей)
3. Профили пользователей хранятся в localStorage и не затрагиваются

## Миграция данных

Для переноса данных на другой компьютер:

1. Используйте функцию «Экспорт раздела» для сохранения профиля в JSON
2. Перенесите файл профиля на новый компьютер
3. Используйте функцию «Импорт раздела» для загрузки профиля

## Известные ограничения

- Размер обрабатываемых файлов ограничен памятью браузера
- Для очень больших файлов (>100 МБ) рекомендуется использовать специализированные инструменты
-localStorage очищается при очистке данных браузера

## Поддержка

При возникновении проблем:

1. Проверьте консоль браузера (F12) на наличие ошибок
2. Убедитесь, что файлы библиотек находятся в папке `js/lib/`
3. Попробуйте открыть приложение в другом браузере

## Лицензия

Приложение разработано для использования государственными органами РФ в рамках подготовки открытых данных для публикации на портале data.gov.ru.
